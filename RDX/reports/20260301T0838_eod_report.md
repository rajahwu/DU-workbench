## Architecture Observations

### What's Working Well

1. **Phase manager → router → React navigation** chain is clean. One `transition()` call drives both engine state and URL.
2. **Packet forwarding pattern**: Each shell reads `dudael:active_packet` from localStorage, enriches it with phase-specific data (vessel choice, drafted cards, level results, door choice), and passes it forward. Data flows through the whole loop.
3. **Core isolation held**: `phases/` has no React imports. All shells import from `@du/phases` via the barrel.
4. **`types.ts` unification** resolved the dual-PhasePacket problem from yesterday's report.

### What Needs Attention

Fixed 1. **Vessel data is hardcoded in SelectShell**: The 5-vessel dataset (200+ LOC including SVG) lives inline in a component. Should migrate to `data/entities/` or a vessels data module.
Fixed 2. **Card pool is hardcoded in DraftShell**: Same — draft cards are inline. Should migrate to `data/` layer.
Fixed 3. **`any` types scattered**: `DraftShell` uses `any` for card types and offerings state. Needs proper `Card` type definitions.

4. **localStorage packet pattern is fragile**: Every shell does `JSON.parse(localStorage.getItem("dudael:active_packet"))` manually. Should use `readActivePacket()` from boot.ts (already exported via barrel).
5. **No state management layer yet**: Redux Toolkit and TanStack Query are installed but unused. Shells read engine state via direct `getRunMeta()` calls and localStorage. Works for prototype but won't scale.

Fixed 6. **TitlePage.tsx exists but isn't routed**: The PixiJS mount wrapper (`TitlePage.tsx`) is orphaned — `main.tsx` routes `/title` to `TitleShell` (HTML/CSS version). Decide which title experience ships.
Fixed 7. **CSS naming inconsistency**: Some shells use `styles.css`, others `style.css` (singular). Minor but worth standardizing. ✅
Fixed 8. **`title-screen.ts` exports `mountTitleScreen`**: The original PixiJS animation now exports a proper mount function with host element + cleanup, but it's only consumed by the unused `TitlePage.tsx`.

9. **DropShell loops to 03_staging**: The `07_drop → 03_staging` transition was added to LEGAL but the original architecture had `07_drop → 04_draft | 01_title`. This is a game design choice, not a bug, but it changes the loop topology.
